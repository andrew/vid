# VID in brief

A VID is a content-addressed identifier for a vulnerability in an open source package, computed by hashing the package name together with the bytes of the file where the bug lives. It exists because automated scanners now find bugs faster than anyone can report them through the CVE process, and many people independently find and report the same bug to the same maintainer. Since the identifier is computed rather than assigned, anyone looking at the same bug gets the same string without asking an authority or knowing the others exist. That gives maintainers a dedup key in their inbox, gives researchers a way to timestamp a finding before disclosing it, and lets consumers match a held set of VIDs against their installed dependency tree before any CVE is published. Two branches, `vid2-function-bytes` and `vid2-function-ast`, explore narrowing the hashed unit from the file to the enclosing function via tree-sitter, trading a parser dependency for stability across unrelated edits to the same file; see [COMPARISON.md](COMPARISON.md).
